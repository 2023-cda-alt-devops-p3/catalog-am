Diagramme de Cas d'Utilisation (Use Case Diagram) :

Vous pouvez l'utiliser pour identifier les acteurs (utilisateurs) 
de votre site web et définir les fonctionnalités que votre site doit offrir aux utilisateurs.
Diagramme de Séquence (Sequence Diagram) :

Il peut être utilisé pour modéliser les interactions entre les utilisateurs 
et votre site web, montrant comment les utilisateurs interagissent avec les fonctionnalités de votre site.
Diagramme de Classe (Class Diagram) :

Si votre site comporte des objets ou des entités importantes, 
ce diagramme peut vous aider à représenter leur structure, leurs attributs et leurs relations.
Diagramme d'État (State Diagram) :

Vous pouvez l'utiliser pour modéliser le cycle de vie des éléments de votre site, 
par exemple, l'état d'une commande (nouvelle, en cours de traitement, terminée, etc.).
Diagramme de Déploiement (Deployment Diagram) :

Si votre site est déployé sur des serveurs ou dans un environnement distribué, 
ce diagramme peut vous aider à modéliser l'infrastructure de déploiement.


Diagramme MERISE : 

MCD (Modèle Conceptuel de Données) :

Diagramme utilisé pour représenter les entités, 
les relations entre les entités et les attributs dans un système d'information. 
Il est similaire au diagramme entité-relation (ERD) dans d'autres méthodologies.

MLD (Modèle Logique de Données) :

Diagramme qui raffine le MCD en ajoutant des détails sur les tables, 
les clés primaires, les clés étrangères, les contraintes d'intégrité, etc. 
Il représente la structure des données dans la base de données.
